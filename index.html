What is Prompt-Driven Development?

PDD is a development methodology where natural language prompts are used to deliberately guide **scaffolding, implementation, testing, refactoring, and documentation** right inside your editor.

The PDD Mindset: Best Practices

Using AI tools responsibly and effectively requires a shift in mindset.

You are the Architect

You own the design and the outcome. The AI is a tool to execute your vision faster, not to create it.

Specificity is King

"Bad Prompt In, Bad Code Out." Be descriptive. The more context you give, the better the output.

Never Trust, Always Verify

The AI is a pair programmer, not an infallible oracle. Review every significant line of generated code.

The Best Prompts are Comments

Writing your intent as a comment first clarifies your own thinking and documents your code for free.

Use AI for the "How"

You define *what* problem to solve. Use the AI to accelerate *how* you implement the solution.

Your Mission

Install an AI assistant, try writing a comment first for one task, and embrace the PDD loop: **Intent -> Generate -> Verify.**
